vet-plus/

Plataforma distribuída com 6 microsserviços Django REST e dashboard React. Cada serviço segue Clean Architecture (domain → application → infrastructure → presentation), banco PostgreSQL isolado e autenticação JWT centralizada, com integração HTTP entre domínios (animais, consultas, vacinas).

ApiDualPersistence/

API REST Express em camadas (routes → controllers → services/models) com persistência híbrida: PostgreSQL para usuários, credenciais e roles; MongoDB para recursos de domínio (carros, motos, marcas, posts). JWT stateless, validação centralizada, Swagger e stack containerizada com Docker Compose.

GrimCodex/

SPA React para guia de Diablo IV acoplada a backend Node/Express. Persistência híbrida: PostgreSQL (auth, perfis) e MongoDB (conquistas, fórum, chat). Módulos de grimório CRUD, fórum com categorias e chat ao vivo via WebSocket.

IronCustom/

Frontend React + Vite para gestão de oficina de motos custom, consumindo a API Dual Persistence via Axios. Módulos de OS (Kanban), CRM, estoque, projetos e financeiro; roteamento protegido por JWT com proxy Vite/Nginx para o backend stateless.

GerenciadorTarefas/

To-do list Flask que aplica 4 Design Patterns GoF: Factory (criação de tarefas), Strategy (ordenação), Observer (eventos) e Facade (interface unificada). API REST + UI Bootstrap; rotas delegam à fachada, mantendo domínio desacoplado da camada HTTP.

JWT/

API Express de referência para autenticação JWT stateless: login gera token assinado, middleware valida claims em rotas protegidas. Estrutura routes/controllers/middlewares demonstra o fluxo completo sem sessão no servidor.

app_kapitour_test/

App mobile React Native/Expo de turismo em Maricá com backend FastAPI em 5 microsserviços (auth, conteúdo, engajamento, commerce, KapiPass). Cada serviço tem SQLite próprio, expostos por nginx como API Gateway; gamificação com XP, carimbos e rankings.

P1_fullstack/

Dashboard full stack Node/Express com dois bancos simultâneos: MySQL (Sequelize) para usuários e MongoDB (Mongoose) para produtos. Arquitetura em camadas (routes → controllers → repositories → models) orquestrada via Docker Compose.
